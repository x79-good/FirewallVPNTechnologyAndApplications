# Lab1：又见面了， HTTP/HTTPS！

## 实验背景

HTTP（HyperText Transfer Protocol，超文本传输协议）是应用层最核心的协议之一。每次打开网页，浏览器与服务器之间就在用 HTTP"对话"。

一次典型的 HTTP 交互分为两部分：

```
浏览器 ──── HTTP 请求 ────▶ 服务器
浏览器 ◀─── HTTP 响应 ──── 服务器
```

**请求报文**结构示例：

```http
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

**响应报文**结构示例：

```http
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...</html>
```

HTTPS 在 HTTP 基础上加入了 TLS 加密，报文内容在传输过程中无法被直接读取。但**浏览器开发者工具**运行在加密之前，可以看到完整的明文请求和响应，是分析 HTTP/HTTPS 协议最方便的入门工具。

1. 用 Chrome 或 Edge 浏览器访问任意 **HTTPS** 站点，例如 `https://www.yxnu.edu.cn/`。

4. 点击列表中第一条请求（通常是页面本身），在右侧查看 **Headers** 标签页，找到 Request Headers 和 Response Headers。
5. 对请求头区域和响应头区域分别**截图**，并按规范命名（见下方截图要求）。
6. 根据截图，完成下方的知识填空。

### C. 知识问答

1. HTTP 请求报文由哪几部分构成？请按顺序列出：

   > 答：请求行 请求头 空行 请求体

2. 状态码 `404` 代表什么含义？状态码 `500` 和 `503` 有什么区别？

   > 答：状态码404代表请求的资源未找到，属于客户端错误，通常是因为请求了一个不存在的路径。状态码500和503虽然都属于服务器错误类别，但含义不同：500代表服务器内部发生了意外错误导致无法完成请求，通常是程序代码报错或配置问题；503代表服务器暂时无法处理请求，通常是由于过载或正在停机维护，这种情况一般是暂时的。

3. GET 与 POST 方法的主要区别是什么？各适用于什么场景？

   > 答：GET 和 POST 的主要区别在于数据传输方式、安全性和用途：
GET 的参数附加在 URL 后面，有长度限制，安全性较低（数据暴露在 URL 中），且会被浏览器缓存。它主要用于获取数据，如浏览文章、搜索查询等只读操作。POST 的参数放在请求体里，无长度限制，相对安全，不会被缓存。它主要用于提交数据，如登录注册、上传文件、表单提交等可能改变服务器状态的操作。

4. HTTP 与 HTTPS 有什么区别？HTTPS 使用了什么机制来保护数据？

   > 答：HTTP与HTTPS的主要区别在于安全性。HTTP协议以明文方式传输数据，不提供任何加密，这意味着通信内容可以被窃听、篡改。HTTPS则是在HTTP的基础上加入了SSL/TLS协议，通过加密保障数据传输的安全。HTTPS主要通过以下几种机制来保护数据：加密机制：对传输的数据进行加密，即使被截获，没有密钥也无法解读内容，防止窃听。身份认证机制：通过SSL/TLS证书验证服务器的身份，确保客户端访问的是真实的网站，防止中间人伪造身份。数据完整性校验：通过消息摘要等方式检测数据在传输过程中是否被篡改，确保数据的完整和准确。

5. 既然 HTTPS 已经加密，为什么浏览器开发者工具仍然能看到请求和响应的明文内容？

   > 答：因为开发者工具显示的是浏览器已经解密之后的数据。
HTTPS 加密保护的是数据在网络中传输的过程。当数据到达你的电脑后，浏览器会负责解密，然后才能渲染网页。开发者工具直接从浏览器内存中读取这些已解密的原始数据，所以能看到明文
